# TalkToMeBaby Docs Site + Repo Sweep Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a hand-authored static HTML docs site in `site/` deployable to Netlify as-is, plus fix repo documentation gaps (README, CHANGELOG, vsix hygiene, extension icon, SECURITY.md, CODE_OF_CONDUCT.md).

**Architecture:** Eleven plain HTML pages sharing one stylesheet and duplicated header/footer chrome — zero build step, zero JS. Content is transcribed from existing verified repo docs, never invented. A pure-CSS karaoke word-sweep on the landing hero demos the product's signature feature.

**Tech Stack:** Plain HTML5 + CSS. Netlify (publish dir only). No frameworks, no JS, no analytics. Fonts: Google Fonts (Bricolage Grotesque display + system stack body).

**Spec:** `docs/superpowers/specs/2026-06-10-docs-site-design.md`

**Model delegation policy (user directive):** All writing is done by lower-cost models — `sonnet` for HTML pages, design system, and icon; `haiku` for mechanical repo fixes. The lead session only dispatches, reviews against sources, and verifies. Each dispatch prompt must be self-contained (agents have zero conversation context).

## Hard content rules (include verbatim in every page-agent prompt)

1. Every factual claim must come from the listed source files. No invented features, numbers, or version strings. Current version: **0.3.0**.
2. Preserve the honest fragility framing: Edge TTS uses an **unofficial** Microsoft endpoint, no SLA, has broken before.
3. Internal links use explicit `.html` paths (works locally and on Netlify; Netlify also resolves extensionless).
4. Use the shared chrome (header/nav/footer) EXACTLY as given — byte-identical across pages except `<title>`, `<meta name="description">`, and the `aria-current="page"` attribute on the active nav link.
5. Semantic HTML, WCAG AA contrast, no JS.

```markdown
# Security Policy

## Supported Versions

Only the latest release (currently 0.3.x) receives security fixes.

## Reporting a Vulnerability

Please report vulnerabilities privately via
[GitHub Security Advisories](https://github.com/rishmadaan/talktomebaby/security/advisories/new).
Do not open a public issue for security reports. You should get a response within a week.

## Scope notes

- **API keys** (ElevenLabs, Sarvam) are stored exclusively via the VS Code
  SecretStorage API (OS keychain). They are never written to settings.json,
  never synced, and never logged. Anything that violates this is a
  security bug — please report it.
- **Edge TTS** uses an unofficial Microsoft endpoint via the `msedge-tts`
  package. Document text is sent to the selected cloud provider during
  synthesis (see the Privacy docs). The macOS `say` provider is fully local.
```

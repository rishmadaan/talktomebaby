Providers

TalkToMeBaby works out of the box on every platform — no API key required to start. The talktomebaby.provider setting defaults to auto: on macOS that resolves to the built-in say provider (official, fully offline); on Windows and Linux it resolves to Edge TTS (free, no key needed). An explicitly chosen provider is always respected.

Provider comparison

Provider Quality Word timing Key required Offline Fragility
macOS say Decent (Enhanced voices) Estimated No Yes Official Apple CLI, stable. macOS only.
Edge TTS Neural, very good Exact (word boundaries) No No Unofficial endpoint. Uses the same endpoint Edge's "Read Aloud" feature uses, via the msedge-tts package. Microsoft offers no contract; it has broken before and could change or get gated at any time. That is the price of free.
ElevenLabs Best-in-class Exact (char alignment) Yes No Official API, stable. Paid.
Sarvam AI Good (Indian English) Estimated Yes No Official API.

Word timing controls how precisely the word-by-word karaoke highlight tracks speech. Exact timing uses data from the provider; estimated timing is inferred from character offsets and synthesis duration.

Per-provider setup

macOS say — default on macOS

Uses the official Apple say command-line tool built into macOS. Audio is synthesized entirely on your machine — no text leaves your device. Nothing to configure: if you are on macOS and leave the provider setting at auto, say is already active.

Novelty voices (Zarvox, Boing, and similar) are filtered out of the voice list. Enhanced voices (downloadable in System Settings → Accessibility → Spoken Content) give noticeably better quality. The say provider is hidden on Windows and Linux.

Edge TTS — default on Windows and Linux

Uses the same speech endpoint that Microsoft Edge's built-in "Read Aloud" feature uses. Free, no API key needed, many voices. Requires an internet connection to Microsoft's speech service.

Default voice: en-US-AriaNeural. You can change it from the gear panel or via TalkToMeBaby: Select Voice in the command palette.

Unofficial endpoint. Edge TTS uses an undocumented Microsoft endpoint. There is no SLA, no public support channel, and it has broken before — the extension already had to migrate from msedge-tts v1 to v2 due to an upstream change. If it breaks again, Windows and Linux users will lose audio until a fix ships. See Roadmap for the plan to replace this with a bundled local TTS engine.

ElevenLabs

High-quality neural voices. Requires an ElevenLabs API key.

To set your key: Cmd+Shift+P → TalkToMeBaby: Set API Key → select ElevenLabs → paste your key. The key is stored via the VS Code SecretStorage API (macOS Keychain, Windows Credential Manager, or libsecret on Linux) — never written to settings.json, never synced, never logged.

Default voice ID: 21m00Tcm4TlvDq8ikWAM. Change it from the gear panel or via TalkToMeBaby: Select Voice.

Free tier note. ElevenLabs free accounts may be blocked from the TTS API and return a 401 error. A paid plan (Starter tier or higher) is recommended for API access.

Sarvam AI

Indian English focus. Requires a Sarvam AI API key.

To set your key: Cmd+Shift+P → TalkToMeBaby: Set API Key → select Sarvam AI → paste your key. Stored the same way as ElevenLabs — SecretStorage, never in settings. Default voice: shubh.

Switching provider and voice

The gear panel inside the Reader is the primary way to change everything. Click the gear (⚙) at the right end of the player bar to open it. The panel slides in above the player bar with:

  • Provider — one row per available provider, each with a short description, a "key required" badge where relevant, and a check (✓) on the active one. Click a row to switch. (macOS say only appears on macOS.)
  • Voice — a dropdown of the active provider's voices, with the current one selected.
  • Appearance — font size stepper (12–28 px) and sentence-band / current-word color pickers, with a Reset to theme option.

Switching keeps your place and never auto-plays. Changing provider or voice reconfigures the reader in place — the Reader is not torn down and rebuilt. The session re-primes paused at your current sentence with the new voice; press play when you're ready. No surprise audio. The settings panel stays open across the switch.

If a key-required provider has no stored key, switching to it prompts for the key first; cancel and the selection snaps back to the current provider.

Command palette (mirrors the gear panel):

  • TalkToMeBaby: Select TTS Provider — pick from a quick-pick list; marks the active provider with ✓ / "(current)". Reconfigures in place on change.
  • TalkToMeBaby: Select Voice — pick a voice for the active provider; same active marker.
  • TalkToMeBaby: Set API Key — store or update an API key for ElevenLabs or Sarvam AI.

The honest part

Edge TTS has no SLA and has broken before. It is the default on Windows and Linux because it is the best free option available right now — good voice quality, exact word timing, no sign-up required. But it runs on an undocumented Microsoft endpoint. The extension already had to migrate between two versions of the underlying package when Microsoft changed the endpoint. If it breaks again and you need audio immediately, your options are: switch to ElevenLabs or Sarvam AI (both have official APIs), or move to macOS.

The real fix is a bundled local neural TTS engine — no unofficial endpoints, no API keys, true offline support on every platform. See Roadmap for the current plan (Piper / Kokoro via ONNX).

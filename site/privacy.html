Privacy

TalkToMeBaby is a local VS Code extension for reading prose files aloud. It does not include analytics, telemetry, tracking pixels, or a remote service operated by this project.

What leaves your machine

When you start reading, TalkToMeBaby turns the current document or selection into text chunks and asks the active text-to-speech provider to synthesize audio. Whether that text leaves your machine depends entirely on which provider is selected.

Provider Text leaves your machine? Notes
macOS say No — fully local Runs locally through Apple's official say command. No text is sent to any network TTS provider. macOS only.
Edge TTS Yes Uses an unofficial Microsoft endpoint (the same service family used by Edge Read Aloud, via the msedge-tts package). Microsoft offers no contract for this; it may change or stop working. No API key required.
ElevenLabs Yes Uses ElevenLabs' official API with your API key.
Sarvam AI Yes Uses Sarvam AI's official API with your API key.

On macOS, the default provider is say — fully offline, no text leaves your machine. On Windows and Linux, the default is Edge TTS, which is a cloud provider using an unofficial endpoint.

Only the text needed for the requested audio chunk is sent. TalkToMeBaby does not intentionally send file paths, repository names, workspace names, API keys, or editor metadata to TTS providers.

Your responsibility: Do not use a network TTS provider for documents you are not allowed to send to that provider. This includes confidential work files, private client data, regulated data, or third-party copyrighted content where transmission to a cloud service is not permitted. See DISCLAIMER.md for broader warranty, liability, and acceptable-use notes.

Where API keys live

ElevenLabs and Sarvam AI keys are stored through the VS Code SecretStorage API, which is backed by the operating system's encrypted credential store:

OS Backing store
macOS Keychain (the same encrypted store Safari uses for passwords)
Windows Windows Credential Manager
Linux libsecret (GNOME Keyring / KWallet)

What never happens

  • Keys are never written to settings.json — they are not stored as plain settings values.
  • Keys are never synced by VS Code Settings Sync.
  • Keys are never logged by the extension.
  • Keys are not visible to other extensions — SecretStorage is namespaced per extension; other extensions cannot read TalkToMeBaby's secrets.
  • The extension has no analytics, telemetry, or tracking. If telemetry is added in a future release, it will respect VS Code's telemetry controls and be documented here before release.

A key lives in memory only while a provider instance is active (during a reading session or a voice-list fetch). Deleting the extension removes access; the OS credential store can also be inspected or cleared directly by the user (for example, via Keychain Access on macOS).

Audio cache

TalkToMeBaby stores synthesized audio chunks on disk so that re-reading the same document costs no additional API credits.

  • Location: VS Code extension global storage — globalStorageUri/audio-cache. This directory is local to your machine and is not synced.
  • Size limit: controlled by talktomebaby.cacheSizeMB, default 200 MB. LRU eviction runs on write when the limit is reached.
  • Contents: cached audio may contain spoken versions of text you asked the extension to read.

The extension also stores the last reading position in VS Code workspace state so it can offer resume-after-reload. This data never leaves your machine.

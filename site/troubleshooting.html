Troubleshooting

TalkToMeBaby v0.3.2 — common problems and their fixes.

Edge TTS produces no audio or a connection error

Edge TTS requires an internet connection to Microsoft's speech service. It uses an unofficial, best-effort endpoint — there is no service-level agreement and it can be unavailable without notice.

If you are offline or the endpoint is unreachable, you have two options:

  • Switch to a cached document — if you have already read the document before, the audio may be served from the disk cache and no network call is needed.
  • On macOS, switch to the built-in say provider: open Settings (Cmd+,), search for TalkToMeBaby: Provider, and set it to say. The say provider is fully offline and never sends text to a network service.

Edge TTS is unofficial. It is free and requires no API key, but it uses an endpoint with no public SLA. If it stops working, switching to a key-based provider such as ElevenLabs is the most reliable alternative. See Providers for a comparison.

macOS say is not available on my machine

The say provider uses the official macOS speech synthesis command. It only works on macOS and is hidden from the provider picker on Windows and Linux.

On Windows and Linux, TalkToMeBaby defaults to Edge TTS, which is free and requires no key. If you want a guaranteed-available alternative with a formal API, set up ElevenLabs or Sarvam AI via Cmd+Shift+P > TalkToMeBaby: Set API Key.

ElevenLabs returns a 401 error

ElevenLabs free accounts may be blocked from the TTS API. A paid plan (Starter tier or higher) is recommended for API access.

Steps to check:

  1. Confirm your API key is stored correctly: Cmd+Shift+P > TalkToMeBaby: Set API Key, select ElevenLabs, and re-enter your key.
  2. Verify the key is valid in the ElevenLabs dashboard. Keys stored by TalkToMeBaby are kept in VS Code SecretStorage (OS keychain — never written to settings.json and never synced).
  3. If your account is on the free tier, upgrade to a paid plan to unlock TTS API access.

The first read of a long document is slow

Audio is synthesized in chunks as you read — subsequent sentences are prefetched in the background while the current one plays. The first chunk must be synthesized before audio can start, which takes a moment for longer documents or slower connections.

Once a chunk is written to the disk cache, replaying it is instant and costs no API credits. The cache limit is 200 MB by default (LRU eviction); you can adjust it with the talktomebaby.cacheSizeMB setting.

The warm-up delay is most noticeable on the very first read. Subsequent reads of the same document are served from cache and start immediately.

Playback highlight is off after I edited the file mid-read

If you edit the document while reading, the session model becomes stale — the word and sentence offsets no longer match the current file contents. TalkToMeBaby detects the change, pauses playback, and shows a prompt offering to restart from your current position or stop.

Use the prompt to restart. Dismissing it and continuing without restarting may result in misaligned highlighting until you restart the session.

Playback stopped after a VS Code window reload

Reloading the VS Code window or restarting the extension host always ends the active session — audio is not preserved across reloads.

TalkToMeBaby saves your position automatically (approximately every 3 seconds while playing). When the extension reactivates, it offers a "Resume?" toast if a session was saved within the last 12 hours. Choosing Resume re-opens the document and jumps straight to your last sentence. Dismissing clears the saved position.

Resume starts at the beginning of the sentence, not mid-sentence. Because audio is not preserved across reloads, there is no way to resume at the exact word. The position is also cleared when you press Stop or when a document finishes naturally.

The saved position is also cleared when you:

  • Press Stop in the player bar.
  • Let the document finish reading naturally.
  • Dismiss the "Resume?" toast.

Still stuck?

If none of the above resolves your issue, open a bug report on GitHub Issues. Include your OS, VS Code version, TalkToMeBaby version (0.3.2), and the active provider. Do not include API keys, tokens, or the text content of private documents in your report.
